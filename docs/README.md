### When to Update

**Always update:**
- Database changes ‚Üí [DATABASE_SCHEMA.md](DATABASE_SCHEMA.md)
- New API endpoints ‚Üí [backend/API_CONTRACT.md](backend/API_CONTRACT.md)

### Style Guide

1. **Use clear headings** - H2 for sections, H3 for subsections
2. **Include code examples** - Show, don't just tell
3. **Add last updated date** - Top of each document
4. **Link between docs** - Use relative paths
5. **Keep it concise** - Developers scan, they don't read
